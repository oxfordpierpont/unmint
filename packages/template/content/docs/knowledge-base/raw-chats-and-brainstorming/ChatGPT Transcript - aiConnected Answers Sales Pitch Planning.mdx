Implementation blueprint (n8n-first, zero fluff)
================================================

**Trigger:** “Answer Published” → WordPress → Webhook (fires when post in category/tag = `answerengine` is published)

**Flow (nodes in order):**

1.  **Webhook (POST /answer-published)** → receives `postId`, `title`, `slug`, `excerpt`, `featuredImage`, `category`, `tags`, `publishedAt`.
2.  **WordPress → Get Post** (by `postId`) → pull full HTML + custom fields: `primary_service_page`, `geo`, `cta_url`.
3.  **Function → Build UTM**
    *   `utm_source` (social|newsletter)
    *   `utm_medium` (linkedin|facebook|gbp|x|email)
    *   `utm_campaign` (`answerengine_${YYYYMM}`)
    *   `utm_content` (slug)
    *   Output canonical `tracked_url`.
4.  **LLM → Generate Social Variations (3x)**
    *   Inputs: title, excerpt, takeaway list (scrape H2s/H3s), `tracked_url`, target persona.
    *   Outputs: `post_short` (plain), `post_value` (1 tip + CTA), `post_question` (engagement).
5.  **Set → Platform Payloads**
    *   LinkedIn (org share text ≤ 3,000 chars)
    *   Facebook Page (message)
    *   Google Business Profile Post (call/book/learn-more + `tracked_url`)
    *   X/Twitter (tight version, ≤280 chars)
6.  **If → Channel Toggles** (env vars: `USE_LINKEDIN`, `USE_FB`, `USE_GBP`, `USE_X`)
7.  **HTTP Request → LinkedIn API** (Organization Shares)
8.  **HTTP Request → Facebook Graph (/\{page-id\}/feed)**
9.  **HTTP Request → GBP API (localPosts.create)**
10.  **HTTP Request → X API (tweets)** _(optional; requires paid creds)_
11.  **Wait → 3 days** → re-post variant #2 per channel
12.  **Wait → 21 days** → re-post variant #3 per channel
13.  **Compose Newsletter**
    *   **LLM → Subject lines (3)**
*   **LLM → Preheader (≤90 chars)**
*   **Function → Build HTML** (template below) with this week’s 3–5 Answers (pull via WordPress “answers last 7 days”)
    14.  **Email Send** (Mailgun/Sendgrid/SES/Gmail) with `newsletter@yourdomain`
15.  **CRM Create/Update Campaign** (HubSpot/Pipedrive/Close via HTTP or native nodes): name = `answerENGINE_${YYYYMM}`, source tags per channel.
16.  **Supabase (or DB) → Upsert `amplify_log`** (post\_id, channel, post\_api\_id, timestamps, utm, status)
17.  **Catch Errors → Notify** (Slack/Email) with payload + retry button.
